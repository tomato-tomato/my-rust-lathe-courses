# Lathe — 提问教程内容

回答读者关于已存储教程某个部分的问题，回答要扎根于该教程实际构建的内容。触发方式：

> **路径解析：** 本技能中所有 `<TUTORIALS_DIR>` 均从 `~/.lathe/config.json` 的 `tutorials_base_path` 字段读取。如果配置文件不存在，使用默认值 `~/others/lathe_tutorials`。

```
/lathe-ask <slug> <part-NN.md>
<读者的问题>
```

解析全部三个要素：slug、部分文件名和问题。

## 协议

1. **阅读该部分**，路径为 `<TUTORIALS_DIR>/<slug>/<part-NN.md>`。当问题跨部分或依赖更早的设置时，阅读同目录下的相邻部分——连续性在这里同样重要。

2. **回答要扎根于本教程的具体产物**——相同的控制性示例、相同的数字、教程使用的相同 voice。读者问的是*他们的*合成器 / *他们的*键值存储，而不是抽象的主题。不要从头重新教授整个主题。

3. **指向教程，不要重新推导。** 优先"看 §3 中的 `process_buffer` 循环——那里的取模在做 X"，而非全新的从零解释。你是站在读者身旁、看着他们正在阅读的页面的向导。

4. **对缺口保持诚实。** 如果问题暴露了教程出错、含糊或用 `[!UNVERIFIED]` 标记的内容，直接说明——不要掩饰。这比自信地挥手更有用。

5. **保持参与**以回答后续问题。这是一场对话，不是一次性回复。

## 边界——只读、对话式

- **本技能不写入任何内容。** 它是纯粹的对话模式。
- **不修改状态：** 不要碰 `metadata.json`、`verify-result.json` 或部分 markdown。不要验证、不要续写、不要打标签。
- 保持回答针对本教程的具体产物，使用其 voice。
