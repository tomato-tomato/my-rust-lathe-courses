# Lathe — 续写教程

为已存储的教程添加下一部分。由 `/lathe-extend <slug> [指导…]` 触发。新部分必须延续*该*教程——它的示例、它的数字、它的 voice——而不是一篇全新的泛化教程。关于结构、研究纪律和标注的所有内容来自 **`lathe`** 技能；阅读并应用它。本技能仅添加续写的不同之处：连续性和"写入后更新"的握手。

> **路径解析：** 本技能中所有 `<TUTORIALS_DIR>` 均从 `~/.lathe/config.json` 的 `tutorials_base_path` 字段读取。如果配置文件不存在，使用默认值 `~/others/lathe_tutorials`。

## 协议

1. **吸收现有教程。** 阅读 `<TUTORIALS_DIR>/<slug>/` 中的每个部分（`part-NN.md`）和 `metadata.json`。在写下一个字之前你需要关键的上下文：
   - **控制性示例**和**固定数字**（采样率、页面大小、缓冲区大小）——精确复用它们。
   - **控制性隐喻**（如有），以及之前的部分是否已告别它。
   - **voice**。从 `metadata.json` 读取 `voice` 字段并应用对应的 voice 规范（内置 voice 规范见 `lathe-voice` 技能）。不要重新选择或漂移 voice；结构和内容仍遵循 `lathe` 技能，它在与 voice 冲突时获胜。
   - **读者的水平**。
   - **仓库和锁定的工具版本**在 `metadata.json` 中（`repo`、`repo_branch`、`tools`）——新部分*继承*它们。基于相同版本写作；不要静默升级到更新的工具链。（如果读者明确想迁移到新版本，这是一个信号——可能是时候创建一个新教程了，而不是在这里悄悄漂移。）
   - 上一部分 **"接下来"** 指向的方向——除非用户的指导重新定向，否则这就是你的任务。

2. **先研究。** 与 `lathe` 技能的"先研究"步骤同样的纪律：实际打开 3-8 个该新部分引入内容的权威来源，在关键事实旁记录 URL，并对每个关键声明有据可依或用 `[!UNVERIFIED]` 标记。

3. **确定下一部分编号。** 从 `metadata.json` 读取现有的 `parts` 数组。新部分是 `part-NN.md`，其中 NN 是下一个序号，零填充到 2 位（例如，如果 `parts` 有 `["part-01.md"]`，下一个是 `part-02.md`）。**检查该名称的部分是否已存在。** 如果存在，告诉用户并停止。

4. **写入该部分**到 `<TUTORIALS_DIR>/<slug>/<part-NN.md>`。
   - 遵循 `lathe` 技能的完整**教程结构**——引子、`## 你将构建什么`、前置条件、具体章节标题、`## 检查点`、`## 接下来`、`## 练习`、`## 来源`。每个部分独立成篇并以检查点结尾。
   - 因为这是**第 N ≥ 2 部分**，以 `> [!RECALL]` 间隔检索节拍开头：一个关于之前部分关键概念的问题。
   - **一个文件，仅此部分。** 不要写 `index.md`，不要碰之前的部分，不要写多个部分。

5. **更新 `metadata.json`。** 读取现有文件，然后更新：
   - 将新部分文件名追加到 `parts` 数组。
   - 将新查阅的来源 URL 追加到 `sources` 数组（去重）。
   - 将 `model` 字段更新为*你*运行的模型（如 `DeepSeek V4`）。
   - 将 `status` 设为 `unverified`。
   - 写回更新后的文件。

6. **告诉用户**已添加——如何查看（直接在 QoderWork 中打开 markdown 文件），以及验证是可选的（`/lathe-verify <slug>`）。然后保持会话以回答后续问题。

## 边界

- 写入新的 `part-NN.md` 和更新 `metadata.json` 是唯一的文件写入操作。
- 不要验证，不要写 `index.md`，不要写多个部分，不要编辑现有部分。

## 保持会话

你仍然是他们的专家向导。保持可用以回答"让这部分更难"、"我们为什么这样组织结构"、"我的检查点做得怎么样"。
